Reptify and Biggener
Reptify and Biggener were two mobile apps completed for a mobile software development class. The objective of Reptify was to become familiar with proper structuring of mobile apps according to a UI-Business-Data layer standard. The goal with Biggener was to utilise the Internet of Things (IoT) to create a mobile app that would allow users to interface with physical reality.

Reptify was made for users to track their gym progress, build routines, calculate mathematically ideal weight ranges for their gym sessions, and time weight repetitions for muscle hypertrophy.

Biggener was a movement optimisation app that used the camera and a neural network to track the movement of the barbell and compare it to the theoretical ideal movement path. It could work in real time or analyse a recorded video saved to the device gallery.

Currently, I am working on a modern consolidated version of the apps for public release.

Technologies: Reptify
- Built with expo using React Native and JavaScript.

- Stores and accesses data with Firebase using JSON.

- Social Media Authentication integration for user specific profiles and data.

- Animated SVG graphics.

- Prioritises ease of use - large buttons, text, and graphics, with optional audio cues for use inside a gym with earphones.

Technologies: Biggener
- Built with Dart and Flutter.

- Accesses user gallery with no storage required.

- Temporary storage with Stateful Widgets.

- Utilises camera access with a real-time object detection neural network for movement tracking.

- Records coordinates, and interpolates between points with a bezier curve.
